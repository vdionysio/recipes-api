Recipes API

Rotas

  • POST /users

    Criar um usuário. Body é necessário.

    Ex: {
      "name": "Vinícius",
      "email": "vini.dionysio@gmail.com",
      "password": "senhasegura"
    }

  • POST /login

    Logar com um usuário. Body é necessário.

    Ex: {
      "email": "vini.dionysio@gmail.com",
      "password": "senhasegura"
    }

  • POST /recipes

    Adicionar uma receita ao banco. Body e um token no header authorization são necessários.

    Ex: {
      "name": "Receitinha boa",
      "ingredients": "Abobrinha, Berinjela, ...",
      "preparation": "cozinar por tantos minutos..."
    }

  • GET /recipes

    Retorna todas receitas armazenadas no banco.

  • GET /recipes/:id

    Retorna uma receita com o id específico passado na requisição.

  • PUT /recipes/:id

    Atualizar uma receita do banco. Body e um token no header authorization são necessários.

    Ex: {
      "name": "Receitinha boa 2.0",
      "ingredients": "Abobrinha, Berinjela,tomate",
      "preparation": "cozinar por muitos minutos..."
    }

  • DELETE /recipes/:id

    Atualizar uma receita do banco. Um token no header authorization é necessário.

  • PUT /recipes/:id/image

    Adicionar uma imagem a receita. Deve ser passada uma imagem num campo image e um token no header authorization.

  • POST /users/admin

    Adicionar um admin. Um body é necessário, além de um token com role de admin.

    Ex: {
      "name": "Vinícius",
      "email": "vini.dionysio@gmail.com",
      "password": "senhasegura"
    }
